WHAT CHANGED IN THIS VERSION
------------------------------
Your project now:
1. Has a brand new "elite" premium look — gold and deep-espresso
   design, smooth animations, elegant fonts (instead of the plain
   dark theme before).
2. Uses a REAL database — PostgreSQL (hosted for free on a service
   called Neon) — instead of a simple file. This means your data is
   safe, permanent, and can handle LOTS of visitors at once.
3. Has a REAL AI agent powering the chat — not just keyword matching.
   It uses OpenRouter (one free API key gives access to many AI
   models) and tries a whole LIST of 13 free models, one after
   another, so if one is busy or asleep, it instantly tries the next
   one. This makes the AI chat properly "agentic" and very hard to
   break. If every single free model somehow fails at once (rare),
   the app quietly falls back to its own built-in keyword-matching
   agent, so the chat never actually breaks for a visitor — it just
   gets a little less smart for a moment.
4. Generates a real PDF invoice automatically the moment an order is
   placed — with a drawn restaurant logo, itemised bill, tax, and
   total, in the same gold/espresso branding as the website. There's
   a "Download PDF Invoice" button right under the order tracker.
5. Lets people SPEAK their order out loud, in Urdu or English, using
   the mic button next to the chat box. Urdu speech is automatically
   translated to English behind the scenes before the AI figures out
   what to order, so the same ordering pipeline works for both
   languages, and it shows you what it "understood" so nothing feels
   hidden.
6. Can SPEAK its replies back out loud (text-to-speech) — click the
   little speaker icon above the chat to turn this on or off. It
   automatically picks the right voice/language for whatever text it's
   reading (English, Urdu, and a few other scripts).
7. Is hardened against common attacks:
   - Every database question is sent in a safe way that blocks
     "SQL injection" (a very common hacking trick).
   - Prices are always double-checked on the server, so nobody can
     trick the site by editing prices in their browser.
   - The server now limits how many requests one visitor can send
     per minute, so it can't be overwhelmed or abused by bots.
   - Extra security headers are switched on automatically (via a
     tool called "Helmet").
   - Your AI key is only ever used on the server, never sent to the
     visitor's browser, so it can't be stolen by anyone viewing the
     page's code.
   Please read this honestly: no website can ever be made "100%
   unhackable" — that promise doesn't exist for any system, anywhere.
   What this update gives you is a properly hardened, professional
   setup that follows the same practices real companies use.

====================================================================
 PART 1 — CREATE YOUR FREE ONLINE DATABASE (NEON)
====================================================================
You only need to do this ONCE. This is the real database that will
store your menu, orders, and inventory forever.

STEP 1 - Create a free Neon account
---------------------------------------
1. Go to this website:  https://neon.tech
2. Click the button that says "Sign Up" (top-right corner).
3. Choose "Continue with GitHub" (easiest) or sign up with email.

STEP 2 - Create your project
---------------------------------
1. After logging in, click "Create a project" (or "New Project").
2. Give it any name, for example: "dastarkhwan".
3. Leave the other settings as they are.
4. Click the button "Create Project".

STEP 3 - Copy your connection string
-----------------------------------------
1. After the project is created, Neon shows you a "Connection String"
   (it starts with "postgresql://...").
2. There is a small copy icon next to it — click it. This copies the
   whole address, including your password. Keep it safe like a
   password, don't share it publicly.
3. Keep this browser tab open — you will paste this in two places
   later (Step 3 of Part 3, and Step 3 of Part 4).

That's it — your free online database is ready and waiting.


====================================================================
 PART 2 — GET YOUR FREE AI KEY (OPENROUTER)
====================================================================
This key is what lets the chat actually understand what people are
typing (instead of just matching exact words). It's free.

STEP 1 - Create a free OpenRouter account
------------------------------------------------
1. Go to this website:  https://openrouter.ai
2. Click "Sign In" (top-right corner).
3. Choose "Continue with Google" or "Continue with GitHub" — whichever
   account you already have is easiest.

STEP 2 - Create your API key
---------------------------------
1. Once logged in, click your account picture/name (top-right corner).
2. Click "Keys" in the menu that appears.
3. Click the button "Create Key".
4. Give it any name, for example: "dastarkhwan".
5. Click "Create".
6. Click the copy icon next to the key that appears. It starts with
   "sk-or-v1-...". Keep it safe like a password, don't share it
   publicly.
7. Keep this tab open — you'll paste this key in two places later
   (Step 3 of Part 3, and Step 4 of Part 4).

That's it — this one free key unlocks every model this project uses.
You don't need to sign up separately for OpenAI, Google, Meta, etc. —
OpenRouter gives you all of them through one key, and this project is
already set up to try 13 different free models in a row for extra
reliability.


====================================================================
 PART 3 — RUN THE PROJECT ON YOUR OWN LAPTOP IN VS CODE
====================================================================

STEP 1 - Install Node.js (only needed once on your laptop)
------------------------------------------------------------
1. Go to this website:  https://nodejs.org
2. Click the big button that says "LTS" (Long Term Support — the
   safe, stable version).
3. Open the downloaded file and click Next -> Next -> Install -> Finish.
4. Restart your laptop if it asks you to.

STEP 2 - Open the project folder in VS Code
----------------------------------------------
1. Open VS Code.
2. Click "File" (top-left corner) then click "Open Folder...".
3. Select the folder named "restaurant-ai-system" and click "Select Folder".
4. You will now see the files (backend, public, etc.) on the left side.

STEP 3 - Create your .env file (this is where your secret keys go)
------------------------------------------------------------------------------
1. In the left sidebar, right-click the "backend" folder.
2. Click "New File".
3. Name it exactly:  .env
   (yes, it starts with a dot, and there is nothing after "env")
4. Open that new empty file and type these two lines inside it:

    DATABASE_URL=paste-your-neon-connection-string-here
    OPENROUTER_API_KEY=paste-your-openrouter-key-here

   Replace each part after the "=" with the values you copied earlier
   (Neon connection string from Part 1, Step 3, and the OpenRouter key
   from Part 2, Step 2). It should look something like:

    DATABASE_URL=postgresql://myname:abc123@ep-cool-name-12345.us-east-2.aws.neon.tech/neondb?sslmode=require
    OPENROUTER_API_KEY=sk-or-v1-abc123yourrealkeyhere

5. Save the file (Ctrl + S).

STEP 4 - Open the Terminal inside VS Code
--------------------------------------------
1. Click "Terminal" in the top menu bar.
2. Click "New Terminal".
3. A dark box opens at the bottom of VS Code — this is where you type commands.

STEP 5 - Go into the backend folder
--------------------------------------
Type this and press Enter:

    cd backend

STEP 6 - Install the project's building blocks
--------------------------------------------------
Type this and press Enter:

    npm install

Wait for it to finish (10-30 seconds). You'll know it's done when your
cursor is blinking again with no more text moving.

STEP 7 - Create your tables and starter menu (only needed once)
---------------------------------------------------------------------
Type this and press Enter:

    npm run setup-db

You should see messages ending with:
    "All done! Your database now has the full starter menu and inventory."

If you instead see a red error mentioning "DATABASE_URL", go back to
Step 3 and double check your .env file has no typos and is saved.

STEP 8 - Start the app
--------------------------
Type this and press Enter:

    npm start

You should see:
    AI Restaurant server running at http://localhost:5000

STEP 9 - Open it in your browser
------------------------------------
1. Hold "Ctrl" on your keyboard and click the link
   "http://localhost:5000" shown in the terminal.
   (If that doesn't open anything, just open Chrome/Edge yourself and
   type that address into the address bar.)
2. You will see the new premium "Dastarkhwan" website with:
   - The Menu on the left
   - The AI Chat in the middle (try typing: "2 pizzas and one coke")
   - Your Order/Cart on the right, with a "Place Order" button

STEP 9b - Try the new voice, speech, and invoice features
------------------------------------------------------------------
1. VOICE ORDERING: Next to the chat box there's a small dropdown
   (اردو / English) and a microphone button. Pick "اردو", click the
   mic button (it turns red and pulses while listening), and speak
   your order out loud. It's automatically translated to English and
   a chat bubble shows you exactly what it understood, before adding
   items to your cart.
   NOTE: Voice input only works in Chrome or Edge (not Firefox/Safari),
   and your browser will ask for microphone permission the first
   time — click "Allow".
2. SPOKEN REPLIES: Click the small speaker icon at the top of the
   chat panel to turn on spoken replies — the AI will then read its
   answers out loud to you. Click it again to mute.
3. PDF INVOICE: After you click "Place Order", a gold "Download PDF
   Invoice" button appears under your order tracker. Click it to see
   a fully-branded, itemised invoice (with a drawn Dastarkhwan logo)
   open in a new tab — ready to save or print.

STEP 10 - How to stop the app
---------------------------------
Click inside the terminal box and press "Ctrl + C".

TIP: Next time you want to run it again, you only need Steps 4, 5
(cd backend) and Step 8 (npm start). You do NOT need to repeat
"npm install" or "npm run setup-db" again.


====================================================================
 PART 4 — PUT THIS LIVE ON THE INTERNET USING VERCEL
====================================================================
This gives you a real link you can send to anyone.

STEP 1 - Create a free GitHub account (skip if you already have one)
--------------------------------------------------------------------
1. Go to https://github.com
2. Click "Sign up" (top-right corner) and follow the steps.

STEP 2 - Upload your project to GitHub
------------------------------------------
Easiest way — using the GitHub Desktop app:
1. Download and install GitHub Desktop from https://desktop.github.com
2. Open GitHub Desktop and log in with your GitHub account.
3. Click "File" -> "Add Local Repository".
4. Click "Choose..." and select your "restaurant-ai-system" folder.
5. If it says "This directory does not appear to be a Git repository",
   click the link that says "create a repository", then click
   "Create Repository".
6. At the bottom-left, type a short message like "First upload" in the
   "Summary" box, then click the blue button "Commit to main".
7. At the top, click the button "Publish repository".
8. Untick "Keep this code private" if you want anyone to see the code
   (optional), then click "Publish Repository".

IMPORTANT: Your .env file (with your database secret) will NOT be
uploaded — this is on purpose and already set up for you, so your
password stays safe. You will type it directly into Vercel instead,
in Step 4 below.

STEP 3 - Start deploying on Vercel
------------------------------------------------
1. Go to https://vercel.com
2. Click "Sign Up" (top-right corner).
3. Choose "Continue with GitHub" and log in with the same GitHub account.
4. Click the button "Add New..." (top-right), then click "Project".
5. Find "restaurant-ai-system" in your list of repositories and click
   the "Import" button next to it.

STEP 4 - Add your secret keys (very important — do this before deploying)
-----------------------------------------------------------------------------------
1. On the settings page Vercel shows you before deploying, look for a
   section called "Environment Variables".
2. In the box labelled "Key" (or "Name"), type:
       DATABASE_URL
3. In the box labelled "Value", paste your Neon connection string
   (the same one from Part 1, Step 3).
4. Click the "Add" button next to it.
5. Now repeat the same two boxes for your AI key:
   Key:   OPENROUTER_API_KEY
   Value: paste your OpenRouter key (the same one from Part 2, Step 2)
6. Click "Add" again.
7. Now click the big black button that says "Deploy".

STEP 5 - Wait for it to finish
------------------------------------
Wait about 30-60 seconds. Confetti appears on the screen when it's done.
Click "Continue to Dashboard".

STEP 6 - Open your live website
------------------------------------
Click the link shown at the top (it looks like
"restaurant-ai-system.vercel.app"). This is your LIVE website link —
copy it and send it to anyone, on any phone or laptop.

Your menu and inventory are already there because you ran
"npm run setup-db" earlier — it filled the same Neon database that
your live site is now using. You do NOT need to run that command again
on Vercel.

STEP 7 - Making changes later
----------------------------------
Whenever you change a file in VS Code and want the live website to
update:
1. Open GitHub Desktop.
2. You'll see your changed files listed automatically.
3. Type a short message in "Summary" (e.g. "Updated menu").
4. Click "Commit to main".
5. Click "Push origin" (top of the window).
Vercel automatically detects this and updates your live link within
about a minute.

 WHY THIS SETUP CAN HANDLE A LOT OF VISITORS
====================================================================
- Vercel automatically creates more copies of your server the moment
  traffic increases, so one busy moment doesn't crash the site.
- Neon (your database) is built the same way — it scales up
  automatically and is designed to be called from many places at once
  without running out of connections.
- The server now compresses its responses (smaller, faster downloads)
  and blocks any single visitor from firing too many requests too fast.
Together, this is a genuinely solid, production-style setup. That
said, "1 million visitors" also depends on which Neon/Vercel pricing
plan you're on — the free plans are generous for demos, projects, and
small-to-medium real traffic, but a truly massive, sudden spike would
need you to upgrade your Neon and Vercel plans too. This project's
code will not be the bottleneck; the plan you pay for is what decides
the ceiling.


====================================================================
 QUICK TROUBLESHOOTING
====================================================================
- "npm is not recognized" -> Node.js is not installed properly. Redo
  Part 3, Step 1, then restart VS Code completely.
- "DATABASE_URL is not set" -> Your .env file is missing, misspelled,
  or not saved. Redo Part 3, Step 3.
- "npm run setup-db" shows a connection error -> Double-check you
  copied the FULL connection string from Neon, with no extra spaces,
  and that it ends in "?sslmode=require".
- Terminal shows a red error after "npm install" -> Close the terminal
  (trash bin icon), open a new one (Terminal -> New Terminal), and
  try "npm install" again.
- Browser shows "This site can't be reached" -> Make sure the terminal
  still shows "AI Restaurant server running..." and you haven't closed it.
- Chat replies feel "dumb" / just keyword-matched instead of smart ->
  Your OPENROUTER_API_KEY is probably missing, misspelled, or not
  saved. Redo Part 3, Step 3, and check for a "AI agent unavailable"
  warning printed in the terminal — it tells you the exact reason.
  This is not a crash — it's the safety-net kicking in on purpose so
  the chat keeps working either way.
- Chat feels slow to reply -> The first free model in the list may be
  busy; the code is automatically trying the next ones in the list of
  13. This normally still finishes in a few seconds.
- Mic button is greyed out / does nothing -> Your browser doesn't
  support voice input. Switch to Chrome or Edge (desktop or Android).
  iPhone Safari does not support this yet.
- Mic button works but nothing happens after speaking -> Check the
  small italic status line under the chat box for the exact reason
  (often a blocked microphone permission — click the padlock/camera
  icon in your address bar and allow the microphone, then reload).
- "Download PDF Invoice" button doesn't appear -> Make sure you
  actually completed "Place Order" first (see the order tracker
  appear) — the button shows up right under it.
- Vercel deploy shows a red "Failed" -> Click on it, scroll to "Build
  Logs" and read the last few red lines. Most often it means the
  DATABASE_URL or OPENROUTER_API_KEY environment variable wasn't
  added — go to your Vercel project -> Settings -> Environment
  Variables, add whichever is missing, then click "Redeploy".
- Live site loads but the menu never appears -> Almost always means
  DATABASE_URL is missing on Vercel, or "npm run setup-db" was never
  run locally against your Neon database. Redo Part 4, Step 7.
